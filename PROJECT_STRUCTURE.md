**Розподіл відповідальностей.**

- **`core.py`** не залежить від графічних бібліотек — це чистий алгоритм, який можна імпортувати будь-де. Містить `polynomial_hash` і `rabin_karp_search` (базові реалізації з конспекту — саме їх розібрано в README рядок за рядком), `rabin_karp_search_all` (усі входження), `rabin_karp_search_recompute` (версія без rolling — для контрасту), інструментовані `polynomial_hash_steps` і `rabin_karp_search_steps` (та сама логіка + **журнали подій** для візуалізацій із лічильниками порівнянь хешів / char-перевірок / колізій / rolling-оновлень), метрики `rabin_karp_metrics`, лічильники char-порівнянь **наївного / KMP / Боєра-Мура** (для підсумкового порівняння чотирьох алгоритмів) та `find_collisions` (пошук колізій скануванням).
- **`style.py`** — єдине джерело візуальної теми: палітра кольорів іменованими константами (зокрема барви хеш-бейджа, символів виходу/входу в rolling, вердиктів 🟢/🟠/🔴 та кривих складності), `FIGURE_DPI`, шрифти та `configure_style()`.
- **`i18n.py`** — двомовні підписи: `t()` повертає рядок поточною мовою (`uk` за замовчуванням / `en`), `set_lang()` перемикає її. Ключ словника — сам український рядок, тож українські рисунки лишаються байт-у-байт незмінними. Рядки-**дані** (текст конспекту, шаблони), **хеш-числа** (90/35/82/6382179) і **літеральний вивід коду** (`Substring found at index 8`) НЕ перекладаються.
- **`visualization.py`** (потребує `matplotlib` і `numpy`) бере палітру зі `style.py` і малює: **поліноміальний хеш** (`draw_polynomial_hash` — сигнатурний кадр «abc → 6382179 → 90»), крок Horner-акумуляції (`draw_hash_build_step`), **ковзне вікно** з хеш-бейджами (`draw_rolling_window`), **rolling-оновлення** (`draw_rolling_update`), порівняння хешів (`draw_hash_compare`), **колізію** (`draw_collision`), еволюцію пошуку (`draw_search_evolution`), rolling проти перерахунку (`draw_rolling_vs_recompute`), складність (`draw_complexity`) і **порівняння чотирьох алгоритмів** (`draw_compare_four`). Текстові підписи й таблиці кроків — `*_step_title`/`*_step_caption`, `search_trace_rows`, `print_*`.
- **`walkthrough.py`** будує покрокові панелі «код ↔ дані» ОКРЕМО для двох функцій: журнали знімків (`build_hash_steps`, `build_search_steps`) із мапою підсвічування рядків коду (`CODE_HASH`/`CODE_SEARCH`), кодова панель (`draw_code_panel`) і композитор (`render_code_step`, `draw_code_walkthrough_grid`); права панель — сітка хешу або ковзне вікно. Колір рядка коду кодує гілку (активний / збіг хешу / підтверджено / колізія / rolling).
- **`animation.py`** зшиває кадри в зациклений **GIF** (через Pillow, завжди) і додатково кодує ту саму послідовність у **MP4** (через `ffmpeg`, якщо доступний). GIF — гарантований формат, MP4 — легший файл із контролами.
- **`examples/_common.py`** прибирає дублювання між прикладами: налаштування `Agg`/шляхів, `save_figure`, `save_anim`, друк теки виводу (`print_saved_location`).
- **`examples/_searches.py`** — єдине джерело правди для всіх навчальних інстансів: `HashCase` (рядок + очікуваний хеш), `SearchCase` (текст + шаблон + позиція) і `CollisionCase` (пара рядків з однаковим хешем), а також дані конспекту `RAW` і `"developer"`. Їх імпортують і приклади, і тести.
- **`examples/06_full_walkthrough.py`** окремо: зберігає всі 21 кадр трасування конспекту (фаза хешу + фаза пошуку) у `docs/images/walkthrough/` і **сам генерує** markdown-блок розбору, вставляючи його в README між маркерами `<!-- WALKTHROUGH:START/END -->` (ідемпотентно; усі числа — з журналів подій).
- Скрипти **`examples/`** лише беруть потрібний інстанс із `_searches.py`, викликають алгоритм і функції малювання та зберігають рисунки.
